Ваш сервис с подпиской решает **фундаментальную проблему**: современные картографические сервисы и Википедия дают *фрагментированную* информацию, но не показывают **связей между прошлым и настоящим**. Вот почему граф знаний создаёт принципиально новый опыт поиска исторической информации:

### 🔗 1. Связность вместо изолированных фактов

| Сервис | Что вы получаете | Ограничение |
|--------|------------------|-------------|
| **Яндекс.Карты / 2ГИС** | «Здание на Тверской, 13 — кафе „Кофейня“» | Нет исторического контекста: кто строил, что было до, какие события происходили |
| **Википедия** | Статья «Тверская улица» с общими фактами | Нет связи с конкретными зданиями: нельзя быстро найти «все доходные дома на Тверской, построенные до 1917» |
| **Ваш граф знаний** | «Здание на Тверской, 13 → построено в 1903 архитектором Ухтомским → до 1917 — доходный дом → в 1920-х — редакция газеты „Известия“ → в 1941 — штаб ПВО → современное кафе» | **Полная временная линия объекта с персоналиями и событиями** |

**Пример запроса через ваш сервис:**
> «Покажи все здания на Арбате, где жили писатели до 1917 года, и какие музеи сейчас находятся в этих зданиях»

→ Карты не ответят. Википедия потребует ручного поиска по 10+ статьям. Ваш граф вернёт ответ за секунду.

### ⏳ 2. Историческая динамика: объекты как процессы, а не точки

Картографические сервисы показывают **статичную реальность**. Ваш граф моделирует **эволюцию города**:

| Возможность | Пример |
|-------------|--------|
| **Изменение названий улиц** | «Горького улица» → «Тверская улица» (1990) → показать все здания, которые «пережили» это переименование |
| **Трансформация функций зданий** | Доходный дом (1900) → Коммуналка (1930) → Офис (1990) → ЖК «Легенда» (2020) |
| **Исчезнувшие объекты** | Храм Христа Спасителя (снесён 1931) → Бассейн «Москва» (1960) → Храм восстановлен (2000) — с привязкой к координатам |
| **Событийная привязка** | «Покажи все здания в радиусе 1 км от Кремля, где проходили заседания Совнаркома в 1918–1922» |

**УТП для подписки:**  
> «Видите современное здание? Нажмите «История» — и увидите, как оно менялось с 1800 года до сегодня, с фотографиями архивов и биографиями владельцев».

### 🧠 3. Семантический поиск через естественный язык (ваши LLM + Cypher)

| Запрос пользователя | Карты | Википедия | Ваш граф |
|---------------------|-------|-----------|----------|
| «Где в Москве можно увидеть архитектуру конструктивизма рядом с метро?» | Требует знания термина «конструктивизм» + ручной фильтрации | Нужно найти статью «Конструктивизм в Москве» → выписать адреса → искать на карте | `Генерация Cypher → MATCH (b:Building {style:'constructivism'})-[:NEAR]->(:MetroStation) RETURN b.address` |
| «Покажи места, связанные с Маяковским в районе Тверской» | Невозможно | Нужно читать биографию → выписывать адреса | `Генерация: (:Person {name:'Маяковский'})-[:LIVED_AT|:WORKED_AT]->(b)-[:LOCATED_ON]->(:Street {name_ru:'Тверская улица'})` |

**Ключевое преимущество:**  
Пользователь не должен знать термины «административный округ», «исторический район» или «архитектурный стиль» — он говорит на естественном языке, а граф сам строит сложные связи.

### 📚 4. Глубина вместо широты: от района до подъезда

| Уровень детализации | Карты | Википедия | Ваш граф |
|---------------------|-------|-----------|----------|
| **Город** | ✓ | ✓ | ✓ |
| **Район** | ✓ | ✓ | ✓ |
| **Улица** | ✓ | ✓ | ✓ |
| **Здание** | ✓ (адрес) | ✗ (редко) | ✓ + год постройки, архитектор, стиль |
| **Подъезд / квартира** | ✗ | ✗ | ✓ (для исторических зданий: «кв. 12 — жил Булгаков в 1921–1924») |
| **Событие в здании** | ✗ | ✗ (только в статьях) | ✓ («в этом зале 15 октября 1941 проходило заседание ГКО») |

**Пример монетизации:**  
> Базовая подписка: поиск по зданиям.  
> Премиум: доступ к данным об отдельных квартирах, владельцах, исторических событиях внутри зданий.

### 🔄 5. Персонализация через подписку: «исторический фид» района

Карты и Википедия — пассивные источники. Ваш сервис с подпиской создаёт **активный исторический фид**:

| Фича | Описание | Монетизация |
|------|----------|-------------|
| **Уведомления по дате** | «Сегодня в 1941 году на этой улице проходил парад» — пуш при геолокации | Базовая подписка |
| **Изменения района** | «В вашем районе обнаружены новые архивные данные о доме №17» | Премиум |
| **Тематические подборки** | «5 зданий модерна рядом с вами» (на основе геолокации + интересов) | Тематические пакеты («Архитектура», «Революция», «Кино») |
| **Семейная история** | «Ваша фамилия упоминается в документах по этому дому 1905 года» (интеграция с архивами) | Премиум + партнерство с архивами |

### 🧩 6. Интеграция разрозненных источников в единую онтологию

| Источник | Проблема | Как решает ваш граф |
|----------|----------|---------------------|
| **Википедия** | Статьи изолированы, нет структурированных связей | Извлечение сущностей → связывание через `(:Person)-[:DESIGNED]->(:Building)` |
| **Архивы Москвы** | Текстовые документы без геопривязки | Геокодирование адресов → привязка к зданиям в графе |
| **OpenStreetMap** | Современные данные, нет истории | Добавление временных слоёв: `(:Building {valid_from: 1900, valid_to: 1930})` |
| **2ГИС / Яндекс** | Коммерческие объекты, нет исторического контекста | Обогащение через связи с историческими событиями |

**Результат:** Пользователь видит не «адрес + рейтинг кафе», а «адрес → историческое здание → архитектор → стиль → события → современное использование».
